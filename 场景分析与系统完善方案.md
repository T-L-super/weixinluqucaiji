## 📊 场景分类

根据微信文章的结构和内容，我们将录取信息分为 3 类场景：

### 场景 1: 结构化表格数据 ✅

**特点:**
- 包含 HTML `<table>` 标签
- 有明确的表头和学生名单
- 每条记录包含：学生姓名、大学、专业等

**示例:**
```
| 学生姓名 | 录取大学 | 专业 |
|---------|---------|------|
| 张三    | 哈佛大学 | 计算机科学 |
| 李四    | 剑桥大学 | 数学 |
```

**处理方式:**
- ✅ 使用 `_extract_from_tables()` 方法
- ✅ 解析表头映射字段
- ✅ 提取每行数据创建记录

**采集成功率:** 95%+

---

### 场景 2: 文本格式录取名单 ⚠️

**特点:**
- 无表格，纯文本描述
- 包含具体学生姓名和大学
- 格式多样（列表、段落等）

**示例:**
```
恭喜以下同学获得名校录取：
张三 - 哈佛大学计算机科学专业
李四 - 剑桥大学数学专业
王五被斯坦福大学物理系录取
```

**处理方式:**
- ✅ 使用 `_extract_from_text()` 方法
- ✅ 正则表达式匹配多种格式
- ✅ 关键词识别（"录取"、"offer"、"被...录取"）

**采集成功率:** 70-80%

---

### 场景 3: 按大学统计的汇总数据 🔴 新增

**特点:**
- 无具体学生姓名
- 按大学统计录取人数
- 通常是文字描述或简单图表

**示例:**
```
2025 届录取成果：
- 哈佛大学：3 人
- 斯坦福大学：5 人
- 剑桥大学：2 枚 offer
- 多伦多大学：10 封录取

或文字描述：
"我校学子共获得美国藤校录取 8 枚，其中哈佛大学 3 枚、耶鲁大学 2 枚..."
```

**当前问题:**
- ❌ 现有提取器无法识别（因为没有学生姓名）
- ❌ 验证逻辑要求 `student_name_cn` 不能为空
- ❌ 丢失了有价值的统计数据

**影响:**
- 大量只公布统计数据的学校喜报无法采集
- 系统缺少宏观统计数据

**修改验证逻辑:**
```python

        return True
    
    # 个人记录：需要姓名和大学
    if not (self.student_name_cn and self.university_cn):
        return False
    
    # ... 其他验证 ...
    return True
```

**新增提取方法:**
```python
def _extract_aggregated_stats(self, text: str) -> List[AdmissionRecord]:
    """提取汇总统计数据"""

    # 模式 3: "获得 X 大学录取 X 枚"
    pattern3 = r'获得 ([\u4e00-\u9fa5A-Za-z\s]+?(?:大学 | 学院)) 录取 (\d+)[\s 人人枚封]*'
    
    # ... 提取逻辑 ...
    
    return records
```

**优点:**
- ✅ 保留完整信息（大学 + 人数）
- ✅ 支持统计分析
- ✅ 不破坏现有数据结构

**缺点:**
- ⚠️ 需要修改数据库表结构
- ⚠️ 需要更新前端展示逻辑

**缺点:**

### 方案 3: 独立统计表（长期方案）

**创建新的数据库表:**
```sql
CREATE TABLE university_stats (
    id INTEGER PRIMARY KEY,
    article_url TEXT,
    article_title TEXT,
    university_cn TEXT,
    university_en TEXT,
    country TEXT,
    student_count INTEGER,
    admission_year INTEGER,
    source_school TEXT,
    created_at TIMESTAMP
);

CREATE INDEX idx_stats_university ON university_stats(university_cn);
CREATE INDEX idx_stats_year ON university_stats(admission_year);
```

**优点:**
- ✅ 数据结构清晰
- ✅ 专门用于统计分析
- ✅ 不影响个人记录

**缺点:**
- ⚠️ 需要创建新表
- ⚠️ 需要额外的 CRUD 操作

### 修改验证逻辑

def is_valid(self) -> bool:
    """验证记录是否有效（放宽要求）"""
    # 情况 1: 有大学信息（允许没有学生姓名，用于汇总数据）
    if self.university_cn and any(kw in self.university_cn for kw in ['大学', '学院', 'University', 'College']):
        # 如果有学生姓名，验证姓名有效性
        if self.student_name_cn:
            if len(self.student_name_cn) < 2 or len(self.student_name_cn) > 5:

        return True
    
    return False

## 🔧 队列管理优化

### 当前问题

1. 所有链接使用相同的提取逻辑
2. 无法区分文章类型
3. 失败率高的文章无法特殊处理

### 优化方案

**1. 添加文章类型标记:**
```python
class CollectionTask:
    # ... 现有字段 ...
    article_type: str = 'unknown'  # unknown/table/text/aggregate
```

**2. 智能提取策略:**
```python
def extract_by_type(self, html: str, article_type: str):
    if article_type == 'table':
        return self._extract_from_tables(html)
    elif article_type == 'aggregate':
        return self._extract_aggregated_stats(html)
    else:
        # 尝试所有方法
        records = self._extract_from_tables(html)
        if not records:
            records = self._extract_from_text(html)
        if not records:
            records = self._extract_aggregated_stats(html)
        return records
```

**3. 失败重试时切换策略:**
```python
# 第一次失败后，标记为其他类型重试
if task.article_type == 'table' and failed:
    task.article_type = 'text'
    retry_extraction()
```
