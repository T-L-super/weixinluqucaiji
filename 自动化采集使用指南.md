# 自动化采集使用指南

## 🎯 功能概述

本系统已完全集成 OpenClaw 采集能力，实现**微信文章 → 自动采集 → 智能识别 → 数据入库 → 后台显示**的全流程自动化。

---

## 📋 使用方式

### 方式 1: 命令行单条采集

```bash
# 采集单篇微信文章
cd /root/.openclaw/workspace/大学录取信息整理系统
python3 openclaw_integration.py https://mp.weixin.qq.com/s/xxxxx
```

**输出示例:**
```
🔄 开始采集：https://mp.weixin.qq.com/s/xxxxx
✅ 采集成功：3 条记录
  - 张明 → 剑桥大学 (自然科学)
  - 李华 → 剑桥大学 (数学)
  - 王芳 → 哈佛大学 (计算机科学)
✅ 保存成功：3 条新增
```

---

### 方式 2: 批量采集

**步骤 1:** 准备任务列表

编辑 `采集任务链接.txt`，每行一个微信文章链接：
```
https://mp.weixin.qq.com/s/G8tuG-VfjdJiplUbUpf4DQ
https://mp.weixin.qq.com/s/n0qZSkqMT1JcfuEuta_KuA
https://mp.weixin.qq.com/s/xxxxx
...
```

**步骤 2:** 执行批量采集
```bash
python3 openclaw_integration.py --batch
```

**输出示例:**
```
📋 找到 13 个采集任务
🔄 开始采集：https://mp.weixin.qq.com/s/xxxxx
✅ 采集成功：5 条记录
...
📊 总计：45 条记录，保存 45 条
```

---

### 方式 3: 管理后台批量导入

**步骤 1:** 访问管理后台
```
http://101.32.98.235:8000
```

**步骤 2:** 进入任务队列
- 点击顶部「📝 任务队列」标签

**步骤 3:** 批量导入链接
- 点击「+ 批量导入」按钮
- 粘贴多个微信文章链接（每行一个）
- 或上传 TXT 文件

**步骤 4:** 自动执行采集
- 系统自动开始采集任务
- 实时显示进度
- 完成后自动更新数据库

## 🔧 高级功能

### 自定义采集参数

编辑 `auto_collector.py` 可配置：

```python
# 并发数（默认 3）
max_concurrent: int = 3

# 超时时间（秒）
timeout=aiohttp.ClientTimeout(total=30)

# 数据库路径
DB_PATH = 'data/admission_system.db'
```

---

### 数据去重策略

系统自动检测重复记录：
- **判断依据:** URL + 学生姓名
- **处理方式:** 跳过已存在记录
- **日志输出:** `⏭️  跳过已存在记录：xxx`

---

### 错误处理

采集失败时：
- 记录错误信息
- 继续处理其他任务
- 最后输出统计报告

**失败原因可能包括:**
- 网络超时
- 文章无法访问
- 无法识别表格/文本
- 数据格式异常

## 📝 注意事项

1. **网络连接:** 确保服务器可以访问微信文章
2. **采集频率:** 建议控制并发数，避免被封禁
3. **数据验证:** 采集后建议人工核对重要信息
4. **字段设置:** 管理后台可自定义显示字段
5. **定期备份:** 建议定期备份数据库文件

## 🆘 故障排查

**问题 1: 采集失败**
```bash
# 检查网络连接
curl -I https://mp.weixin.qq.com

# 测试单条采集
python3 openclaw_integration.py https://mp.weixin.qq.com/s/xxxxx
```

**问题 2: 数据未显示**
```bash
# 检查数据库
sqlite3 data/admission_system.db "SELECT COUNT(*) FROM admission_records"

# 重启后端服务
pkill -f uvicorn
cd backend && python3 -m uvicorn app.main:app --host 0.0.0.0 --port 8000 &
```

**问题 3: 提取结果为空**
- 检查文章格式（是否包含表格）
- 查看日志输出（提取器详细信息）
- 手动测试提取器
