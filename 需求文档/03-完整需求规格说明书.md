## 1. 项目概述

### 1.1 项目背景

随着中国国际化教育的快速发展，各国际学校、国际部每年产生大量海外大学录取数据。这些数据分散在各个学校的微信公众号、官网喜报中，缺乏统一的收集、整理和分析平台。

**贝优 - 大学录取信息整理系统** 旨在通过自动化采集和智能提取技术，聚合全互联网上各个高中学生的升学案例数据，为家长、学生、教育机构提供权威的升学参考。

### 1.2 项目目标

| 目标类型 | 具体目标 | 衡量指标 |
|---------|---------|---------|
| **数据采集** | 自动化采集微信文章 | 支持 1000+ URL 批量采集 |
| **数据提取** | 智能识别录取信息 | 表格提取准确率 > 95% |
| **数据管理** | 结构化存储和查询 | 支持 10 万 + 记录高效查询 |
| **数据分析** | 多维度统计分析 | 提供 10+ 统计维度 |
| **数据导出** | 灵活的数据输出 | 支持 Excel/CSV/PDF 格式 |

### 1.3 核心价值

1. **数据聚合** - 分散信息集中管理
2. **智能提取** - AI 自动识别结构化
3. **统计分析** - 多维度数据洞察
4. **趋势预测** - 升学规律分析

### 1.4 目标用户

| 用户类型 | 核心需求 | 使用场景 |
|---------|---------|---------|
| **学生家长** | 选校参考、录取概率评估 | 查询目标学校历年录取数据 |
| **教育机构** | 市场分析、竞品调研 | 统计各机构录取成果 |
| **学校老师** | 升学指导、案例参考 | 获取同类学生录取案例 |
| **研究人员** | 教育研究、趋势分析 | 导出数据进行学术研究 |

```
│                 │      │                 │      │                 │
│ - 创建采集任务  │      │ - collection_   │      │ - 每 30 秒轮询    │
│ - 查看任务队列  │      │   tasks 表      │      │ - 执行采集      │
│ - 监控进度      │      │ - 状态更新      │      │ - 回写结果      │
└─────────────────┘      └─────────────────┘      └─────────────────┘
```

### 8.2 工作流程

```
1. 用户在管理后台创建采集任务
           ↓
2. 任务写入 collection_tasks 表 (status=pending)
           ↓
3. OpenClaw 轮询器定期查询待处理任务
           ↓
4. 发现 pending 任务，更新状态为 processing
           ↓
5. 调用采集器执行微信文章采集
           ↓
6. 提取录取信息并保存到 admission_records 表
           ↓
7. 更新任务状态为 completed/failed
           ↓
8. 管理后台自动刷新显示结果
```

### 8.3 轮询器配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 轮询间隔 | 30 秒 | 查询任务队列的频率 |
| 并发任务数 | 3 | 同时处理的任务数 |
| 最大重试次数 | 2 | 失败任务重试次数 |
| 超时时间 | 30 秒 | 单个任务超时时间 |
