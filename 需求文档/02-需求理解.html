1. 项目概述

🎯 核心目标

建立一套完整的大学录取信息管理系统，从微信公众号文章自动采集各国际学校的录取喜报，提取结构化数据，提供查询、统计、导出等功能。

关键价值： 将分散在各学校公众号的录取信息集中管理，形成可查询、可统计、可分析的数据库，为升学指导提供数据支持。

2. 用户角色分析

角色 职责 核心需求
系统管理员 系统维护、用户管理 任务队列管理、系统监控、权限控制
升学指导老师 数据查询、统计分析 按大学/国家/专业查询录取情况、生成统计报表
数据采集员 添加采集任务、审核数据 批量导入链接、监控采集进度、审核提取结果
学校领导 查看统计报告 年度升学报告、录取趋势分析、对比分析

3. 核心功能模块

3.1 数据采集模块

📥 功能描述

负责从微信文章链接中提取录取信息，包括文章抓取、内容解析、数据提取、数据验证等环节。

关键特性

  • 支持微信文章链接批量导入（Excel/手动）
  • OpenClaw 自动抓取和解析
  • 智能提取学生、大学、专业、奖学金等信息
  • 数据验证和去重
  • 支持人工审核和补充

3.2 任务队列模块

⚙️ 功能描述

管理采集任务队列，支持 1000+ 任务的调度和执行。

关键特性

  • 任务优先级管理（1-10 级）
  • 并发控制（可配置并发数）
  • 失败自动重试（指数退避）
  • 定时任务调度
  • 实时进度监控
  • 任务状态追踪（待处理/处理中/完成/失败）

3.3 数据存储模块

💾 功能描述

结构化存储录取信息，支持高效查询和统计。

核心数据表

  • admission_records - 录取记录表（核心）
  • collection_tasks - 采集任务队列表
  • universities - 大学信息表
  • countries - 国家/地区表
  • source_schools - 来源学校表
  • admission_requirements - 录取条件详情表
  • statistics_daily - 每日统计表

3.4 管理后台模块

🖥️ 功能描述

Web 界面进行数据查询、统计、导出和系统管理。

核心页面

  • 仪表盘 - 总体统计、实时数据
  • 采集队列 - 任务列表、进度监控
  • 录取数据 - 数据查询、编辑、审核
  • 统计分析 - 多维度统计图表
  • 数据导出 - Excel/CSV/PDF导出
  • 系统设置 - 用户、权限、配置

5. 工作流程

1. 添加采集任务
→
2. 任务进入队列
→
3. OpenClaw 采集
→
4. 数据提取
→
5. 写入数据库
→
6. 更新统计

流程说明：

1. 用户通过管理后台批量导入微信文章链接到采集队列

2. 系统为每个链接创建采集任务，设置优先级和状态

3. OpenClaw 从队列获取任务，抓取微信文章内容

4. AI 解析文章，提取学生、大学、专业等结构化信息

5. 验证数据完整性，去重后写入数据库

6. 更新任务状态，触发统计数据更新

6. 技术选型

🔧 推荐技术栈

后端框架 FastAPI + Python 3.11+
任务队列 Redis + RQ / Celery
数据库 MySQL 8.0+
前端框架 Vue 3 + Element Plus
实时推送 WebSocket
部署 Docker + Docker Compose

7. 开发计划

开发模式： AI 团队并行开发

Agent 1： 数据库开发（建表、初始化、测试数据）

Agent 2： 后端 API 开发（FastAPI 框架、接口实现）

Agent 3： OpenClaw 集成（采集器、提取器、队列）

Agent 4： 前端开发（管理后台界面、图表）

Agent 5： 测试部署（单元测试、Docker 配置、文档）

总开发周期： 2-3 小时 ⚡（AI 并行开发）

里程碑： 1 小时完成核心功能，3 小时全功能上线

8. 风险与注意事项

⚠️ 技术风险

  • 微信文章反爬机制可能导致采集失败
  • 文章格式不统一影响提取准确率
  • 大量并发任务可能影响系统性能

🛡️ 应对措施

  • 实现智能重试和失败降级机制
  • 建立人工审核和补充流程
  • 采用异步任务队列和限流策略
  • 定期备份数据库，建立恢复机制
