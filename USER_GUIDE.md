## 系统简介

### 什么是大学录取信息整理系统？

大学录取信息整理系统是一款专为教育机构、招生办公室设计的信息管理工具，帮助您高效地收集、整理、查询和统计大学录取信息。

### 核心功能

- 📝 **录取记录管理** - 添加、编辑、删除录取信息
- 🔍 **智能查询** - 多条件筛选和搜索
- 📊 **数据统计** - 可视化统计报表
- 📥 **批量导入** - 支持 Excel、CSV 格式导入
- 📤 **数据导出** - 导出为 JSON、CSV 格式
- 🔄 **实时更新** - WebSocket 实时推送任务进度

### 技术特点

- **单端口部署** - 仅需 80 端口，简化运维
- **前后端一体** - FastAPI 统一服务
- **轻量级** - 基于 SQLite（开发）/ MySQL（生产）
- **易扩展** - Docker 容器化部署

## 功能说明

### 📝 录取记录管理

#### 添加记录

1. 点击页面右上角「+ 添加记录」按钮
2. 填写以下信息：
   - **学生姓名**（必填）
   - **学号**（必填，唯一）
   - **录取学校**（必填）
   - **专业**（必填）
   - **录取类型**（统招/自主招生/其他）
   - **分数**（可选）
   - **省份**（可选）
   - **年份**（可选）
   - **备注**（可选）
3. 点击「保存」

#### 编辑记录

1. 在记录列表中点击要编辑的记录
2. 修改相关信息
3. 点击「更新」保存修改

#### 删除记录

1. 在记录列表中点击要删除的记录
2. 点击「删除」按钮
3. 确认删除操作

⚠️ **注意**：删除操作不可恢复，请谨慎操作！

---

### 🔍 查询与筛选

#### 快速搜索

在搜索框中输入关键词，系统会自动搜索：
- 学生姓名
- 学号
- 学校名称
- 专业名称

#### 高级筛选

点击「筛选」按钮，可以按以下条件筛选：

| 筛选条件 | 说明 |
|---------|------|
| 录取状态 | 待处理/已确认/已拒绝 |
| 学校 | 选择特定学校 |
| 专业 | 选择特定专业 |
| 年份 | 选择录取年份 |
| 省份 | 选择生源地 |
| 分数范围 | 设置最低/最高分 |

#### 排序

点击表头可以按该列排序：
- 点击一次：升序
- 点击两次：降序
- 点击三次：取消排序

### 📥 批量导入

#### 支持格式

- **Excel** (.xlsx, .xls)
- **CSV** (.csv)
- **文本** (.txt) - 特定格式

#### Excel/CSV 模板

下载模板文件，按以下格式填写：

| 学生姓名 | 学号 | 录取学校 | 专业 | 录取类型 | 分数 | 省份 | 年份 |
|---------|------|---------|------|---------|------|------|------|
| 张三 | 2026001 | 北京大学 | 计算机科学 | 统招 | 650.5 | 北京 | 2026 |

1. 点击「导入」按钮
2. 选择文件

5. 查看导入结果

#### 导入结果

导入完成后显示：
- ✅ 成功数量
- ⚠️ 重复数量（学号已存在）
- ❌ 失败数量（格式错误）

### 最佳实践

#### 数据录入

✅ **推荐做法**：
- 使用批量导入处理大量数据
- 定期备份重要数据
- 使用统一的命名规范
- 及时更新录取状态

❌ **避免做法**：
- 手动逐条录入大量数据
- 使用不规范的学号格式
- 长期不更新记录状态
- 不定期备份数据

#### 数据查询

✅ **推荐做法**：
- 使用组合条件精确筛选
- 保存常用筛选条件
- 定期导出重要数据

❌ **避免做法**：
- 使用过于宽泛的搜索条件
- 同时应用过多筛选条件

## 常见问题

### Q1: 忘记学号怎么办？

**A**: 可以使用姓名、学校等其他信息进行查询。

### Q2: 导入时发现重复学号怎么办？

**A**: 系统会自动跳过重复的学号，并在导入报告中显示。如需更新已有记录，请先删除旧记录或手动编辑。

### Q3: 如何修改已确认的录取记录？

**A**: 点击记录进入编辑模式，修改后保存即可。系统会记录修改历史。

### Q4: 数据可以恢复吗？

**A**: 删除的记录无法直接恢复。建议定期导出数据备份。

### Q5: 支持多用户同时使用吗？

**A**: 支持。系统采用数据库事务机制，确保数据一致性。

### Q6: 最大支持多少条记录？

**A**: 
- SQLite（开发）：推荐 10 万条以内
- MySQL（生产）：支持百万级以上

### Q7: 如何在手机上使用？

**A**: 系统采用响应式设计，支持手机浏览器访问。

### Q8: 数据安全性如何保障？

**A**: 
- 本地部署，数据存储在本地
- 支持数据库加密
- 定期备份建议

### 数据字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 学生姓名 | 文本 | 是 | 2-50 个字符 |
| 学号 | 文本 | 是 | 唯一标识，不能重复 |
| 录取学校 | 文本 | 是 | 学校全称 |
| 专业 | 文本 | 是 | 专业名称 |
| 录取类型 | 文本 | 否 | 统招/自主招生/其他 |

| 省份 | 文本 | 否 | 生源地 |
| 年份 | 数字 | 否 | 录取年份 |
| 状态 | 文本 | 否 | pending/confirmed/rejected |
| 备注 | 文本 | 否 | 补充说明 |

### 状态说明

| 状态 | 英文 | 说明 |
|------|------|------|
| 待处理 | pending | 新录入，待确认 |
| 已确认 | confirmed | 已核实确认 |
