## 一、功能概述

在任务队列列表页面新增"录取信息处理状态查看与触发"功能，支持：
1. 从 MySQL 数据源中筛选"大学录取相关"文章数据
2. 按时间段查看数据处理状态
3. 管理员手动设置筛选条件并触发任务
4. 通过 URL 比对判定数据是否已处理

## 二、开发内容

### 2.1 后端 API 开发

#### 文件列表

| 文件 | 说明 |
|------|------|
| `backend/app/mysql_connection.py` | MySQL 数据库连接配置 |
| `backend/api/admission_status.py` | 录取信息状态查询 API |
| `backend/app/main.py` | 注册新 API 路由 |

#### API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/admission-status/status` | GET | 获取录取信息处理状态（按月聚合） |
| `/api/admission-status/articles` | GET | 获取指定月份的详细文章列表 |
| `/api/admission-status/process` | POST | 触发录取信息处理任务 |

#### API 请求示例

**获取状态**
```
GET /api/admission-status/status?start_time=2026-04-01%2000:00:00&end_time=2026-04-30%2023:59:59&keywords=录取,offer
```

**响应示例**
```json
{
  "data": [
    {
      "month": "2026-04",
      "status": "partial",
      "total": 142,
      "processed": 2,
      "unprocessed": 140
    }
  ],
  "start_time": "2026-04-01 00:00:00",
  "end_time": "2026-04-30 23:59:59",
  "keywords": ["录取", "offer"]
}
```

## 五、使用说明

### 5.1 访问页面

1. 打开任务队列页面：`/tasks`
2. 页面顶部显示"录取信息处理状态"面板

### 5.2 查询数据

1. 设置开始时间（默认：2026-04-01 00:00:00）
2. 设置结束时间（默认：当前时间）
3. 设置关键词（默认：录取,offer）
4. 点击"查询"按钮

### 5.3 查看详情

1. 在月份列表中找到目标月份
2. 点击"查看"按钮
3. 在弹出的对话框中查看文章列表

### 5.4 触发处理

1. 在月份列表中找到未处理或部分处理的月份
2. 点击"触发处理"按钮
3. 在确认对话框中点击"确认处理"
4. 系统将自动处理未处理的数据

## 六、状态说明

| 状态 | 颜色 | 说明 |
|------|------|------|
| 已处理 | 绿色 | 该月数据全部已处理 |
| 部分处理 | 橙色 | 该月数据部分已处理 |
| 未处理 | 红色 | 该月数据均未处理 |
| 无数据 | 灰色 | 该月没有命中数据 |

## 七、注意事项

1. **权限控制**: 该功能仅管理员可用
2. **重复处理**: 系统会自动跳过已处理的数据
3. **关键词匹配**: offer 匹配不区分大小写
4. **时间范围**: 支持自定义时间范围查询
