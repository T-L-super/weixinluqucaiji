3. 技术栈更新

组件 技术选型 说明
数据库（开发） SQLite 3 本地 轻量级，无需配置，适合本地开发
数据库（生产） MySQL 8.0+ 线上 高性能，支持并发，适合生产环境
ORM 框架 SQLAlchemy 2.0 支持 SQLite 和 MySQL，无缝切换
后端框架 FastAPI + Python 3.11+ 高性能异步，自动文档
任务队列 Redis + RQ 轻量级，易部署
前端框架 Vue 3 + Element Plus 组件丰富，开发效率高
部署 Docker + Docker Compose 容器化，一键部署

4. 数据库配置（双模式）

开发环境（SQLite）

# .env.development DATABASE_URL=sqlite:///./admission_system.db DATABASE_TYPE=sqlite

生产环境（MySQL）

# .env.production DATABASE_URL=mysql://root:password@localhost/admission_system DATABASE_TYPE=mysql

💡 优势： 开发阶段使用 SQLite 无需安装 MySQL，降低开发门槛；生产环境切换 MySQL 保证性能和并发能力。SQLAlchemy ORM 自动适配，代码无需修改。

🎯 上线时间：今天内

6. 风险控制

⚠️ 潜在风险

  • 微信文章反爬机制可能影响采集成功率
  • 文章格式多样性可能降低提取准确率
  • 多 Agent 并行可能导致代码风格不一致

🛡️ 应对措施

  • 实现智能重试和代理池机制
  • 建立人工审核和补充流程
  • 统一代码规范和自动格式化（Black + ruff）
  • 集成测试确保各模块协同工作
