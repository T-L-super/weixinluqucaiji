## 一、需求背景

当前需要在任务队列列表页面中新增一项处理功能，用于从指定 MySQL 数据源中筛选"大学录取相关"文章数据，并在页面中按时间段查看这些数据是否已处理，同时支持管理员手动触发后续处理任务。

本次需求重点不是自动处理，而是：
- 在页面中按时间段查看数据处理状态
- 支持管理员手动设置筛选条件并触发任务
- 判断某条数据是否已处理的依据，是其 url 是否已存在于本地 SQLite 表中

---

## 二、需求目标

在任务队列列表页面新增一个模块，支持：

1. 从 MySQL 表 `bestieu_test2.wechat_official_account_articles` 中读取数据
2. 根据标题和时间条件筛选目标文章
3. 按月切分时间段展示
4. 展示每个月对应数据是否已处理
5. 管理员可在页面上修改筛选条件
6. 管理员可手动触发某个时间范围内的数据处理任务

## 四、筛选规则

### 4.1 默认筛选条件

从 `bestieu_test2.wechat_official_account_articles` 表中读取数据时，默认按以下条件筛选：

#### 标题条件

`title` 字段包含以下任一关键词：
- `录取`
- `offer`

要求：
- 对 `offer` 的匹配需不区分大小写
- 默认先按这两个关键词匹配
- 页面上允许管理员后续手动修改关键词条件

#### 时间条件

`create_time >= 2026-04-01 00:00:00`

要求：
- `create_time` 字段类型为 DATETIME
- 页面上允许管理员选择时间范围
- 初始默认时间为：2026-04-01 00:00:00 起

### 4.2 备注

- 当前无需考虑额外误匹配排除逻辑
- 当前数据量较小，约 500 条
- 源表已有 `create_time` 索引，可直接使用

## 五、"已处理"判定规则

### 5.1 判定逻辑

源表中的每条文章记录，以其 `url` 字段作为唯一比对依据。

若该 url 已存在于 SQLite 表 `admission_records` 的 `article_url` 字段中，则该文章视为**已处理**。

### 5.2 状态定义

| 状态 | 定义 |
|------|------|
| 已处理 | url 存在于 `admission_records.article_url` 中，且表示该数据已完成处理 |
| 未处理 | url 不存在于 `admission_records.article_url` 中 |

### 5.3 时间段状态定义

按月汇总后，每个月展示该时间段的处理状态。

建议状态定义如下：

| 状态 | 定义 |
|------|------|
| 已处理 | 该月命中的数据全部已处理 |
| 部分处理 | 该月命中的数据部分已处理 |
| 未处理 | 该月命中的数据均未处理 |
| 无数据 | 该月没有命中数据 |

## 七、任务触发要求

### 7.1 触发方式

**手动触发** - 由管理员在页面选择条件后点击触发

### 7.2 触发范围

管理员可基于当前页面选择的条件触发任务，包括：

- 指定关键词条件

### 7.3 触发目标

触发后，系统应对符合条件且未处理的数据执行后续处理流程。

### 7.4 处理结果落库

处理完成后的结果写入：
- SQLite 数据库：`/root/.openclaw/workspace/大学录取信息整理系统/data/admission_system.db`
- 表：`admission_records`

## 八、查询与处理逻辑说明

### 8.1 源数据查询逻辑

从 MySQL 查询符合条件的数据：
- 时间范围命中
- title 包含关键词"录取"或"offer"
- offer 大小写不敏感

### 8.2 状态比对逻辑

将查询出的源数据按 url 与 SQLite 中 `admission_records.article_url` 做比对：
- 命中则视为已处理
- 未命中则视为未处理

### 8.3 时间段聚合逻辑

将命中的源数据按 `create_time` 的月份聚合，输出每个月的处理状态。

## 九、建议 SQL 逻辑

### 9.1 MySQL 查询示意

```sql
SELECT
    id,
    title,
    url,
    create_time
FROM bestieu_test2.wechat_official_account_articles
WHERE create_time >= '2026-04-01 00:00:00'
  AND (
      title LIKE '%录取%'
      OR LOWER(title) LIKE '%offer%'
  );
```

如果页面支持时间区间，则改为：

```sql
SELECT
    id,
    title,
    url,
    create_time
FROM bestieu_test2.wechat_official_account_articles
WHERE create_time >= :start_time
  AND create_time < :end_time
  AND (
      title LIKE '%录取%'
      OR LOWER(title) LIKE '%offer%'
  );
```

### 9.2 SQLite 已处理数据查询示意

```sql
SELECT article_url
FROM admission_records;
```

或按 URL 集合批量比对。
