3. 核心模块设计

3.1 任务队列模块

设计要点

  • 使用 Redis 作为任务队列存储
  • 支持优先级队列（1-10 级）
  • 实现任务状态机（待处理→处理中→完成/失败）
  • 支持失败重试（指数退避策略）
  • 支持定时任务调度
# 任务队列示例代码 from rq import Queue from redis import Redis redis_conn = Redis(host='localhost', port=6379) queue = Queue('admission_collection', connection=redis_conn) # 添加高优先级任务 queue.enqueue( process_article_task, article_url='https://...', task_id=1001, priority='high' ) # 后台 Worker 执行 # rq worker admission_collection

3.2 OpenClaw 采集模块

工作流程

  1. 从任务队列获取待处理任务
  2. 使用 curl 抓取微信文章 HTML
  3. 解析 HTML 提取关键信息（标题、描述、正文）
  4. 使用正则和 AI 提取结构化录取数据
  5. 数据验证和清洗
  6. 写入数据库
  7. 更新任务状态

3.3 数据提取模块

提取策略

  • 表格提取： 识别 HTML 表格，解析表头和数据行
  • 文本提取： 正则匹配"X 同学录取 X 大学"模式
  • 关键词提取： 识别奖学金、专业等关键词
  • AI 辅助： 对复杂格式使用 AI 理解提取

3.4 数据验证模块

验证规则

  • 必填字段检查（学生姓名、大学、年份）
  • 国家名称标准化（映射到标准国家表）
  • 分数合理性检查（TOEFL 0-120, SAT 400-1600）
  • 重复数据检测（学生 + 大学 + 年份唯一）
  • 数据完整度评分（1-5 分）

4. 数据库设计

核心表： 7 张主表 + 3 张辅助表

索引策略： 常用查询字段建立索引，复合索引优化组合查询

分区策略： 录取记录表按年份分区，提升查询性能

表名 说明 主要字段
admission_records 录取记录表（核心） 学生、大学、专业、奖学金等
collection_tasks 采集任务队列 URL、状态、优先级、结果
countries 国家/地区表 国家名、洲、地区
universities 大学信息表 大学名、排名、类型
source_schools 来源学校表 学校名、公众号、联系人
admission_requirements 录取条件详情 GPA、语言、标化要求
statistics_daily 每日统计表 日期、学校、统计数据

6. 管理后台设计

页面列表

  • 仪表盘： 总体统计、实时数据、快捷操作
  • 采集队列： 任务列表、批量操作、进度监控
  • 录取数据： 数据查询、编辑、审核、导出
  • 统计分析： 多维度图表、趋势分析、对比分析
  • 系统设置： 用户管理、权限配置、系统参数

关键交互：

• 批量导入：支持 Excel 拖拽上传，实时解析预览

• 实时推送：WebSocket 推送任务进度，无需手动刷新

• 高级筛选：多条件组合筛选，保存筛选方案

• 数据导出：自定义导出字段，支持多种格式

7. 并发与性能

并发策略

  • 任务队列支持多 Worker 并发处理
  • 数据库连接池管理（最大连接数 50）
  • Redis 缓存热点数据（大学列表、国家列表）
  • 异步任务处理，不阻塞主线程

性能优化

  • 数据库查询优化（索引、分页、避免 N+1）
  • 静态资源 CDN 加速
  • 前端懒加载和虚拟滚动
  • 统计数据预计算和缓存

8. 安全设计

认证： JWT Token 认证，支持刷新 Token

授权： RBAC 角色权限控制（管理员/普通用户）

审计： 操作日志记录，支持追溯

备份： 数据库每日自动备份，保留 30 天
